# Changelog - היסטוריית שינויים

## [2.0.0] - 2025-02-15

### ✨ תכונה חדשה: לוגים נקיים (Clean Logs)

הוספת מערכת לוגים חדשה שמתעדת **רק** את השיחות עם Cursor AI, ללא debugging ומידע טכני.

#### שינויים בקוד:

1. **קלאס חדש: `CleanLogger`**
   - מיקום: `cron_agent.py`
   - תפקיד: יצירה וניהול של לוגים נקיים
   - פיצ'רים:
     - יצירת קובץ לוג יומי: `conversation_YYYY-MM-DD.log`
     - פורמט נקי וקריא
     - כולל timestamp ו-task ID

2. **עדכון `CursorAgent`**
   - הוספת פרמטר `clean_logger` ל-constructor
   - עדכון מתודת `execute()` לכתיבה ללוג הנקי
   - שמירת פרומפט + תשובה בלבד

3. **עדכון `CronAgent`**
   - אתחול `CleanLogger` אוטומטי
   - הצגת מיקום הלוג בהפעלה
   - העברת task_id לצורך logging

#### קבצים חדשים:

📁 **תיקייה ותיעוד:**
- `clean_logs/` - תיקיית הלוגים הנקיים
- `clean_logs/README.md` - מדריך לתיקייה
- `clean_logs/conversation_example.log` - דוגמת לוג
- `CLEAN_LOGS_GUIDE.md` - מדריך מקיף

🔧 **כלי עזר:**
- `view_clean_logs.sh` - סקריפט bash לצפייה בלוגים
  - תפריט אינטראקטיבי
  - 6 אפשרויות צפייה/חיפוש
  - מעקב חי (tail -f)

- `analyze_clean_logs.py` - סקריפט Python לניתוח
  - סטטיסטיקות כלליות
  - ניתוח זמני (לפי יום/שעה)
  - זיהוי סוגי תשובות
  - ניתוח פרומפטים
  - ייצוא סיכום

📊 **עדכוני תיעוד:**
- `README.md` - הוספת סקשן על לוגים נקיים
- `presentation_outline.md` - 4 שקפים חדשים על לוגים
- `requirements.txt` - ללא שינוי (כל התלויות קיימות)

#### שימוש:

```bash
# הפעלה רגילה - לוג נקי נוצר אוטומטית
python cron_agent.py

# צפייה בלוג
./view_clean_logs.sh

# ניתוח לוגים
python analyze_clean_logs.py
```

#### דוגמת פורמט לוג נקי:

```
======================================================================
[2025-02-15 14:30:00] Task ID: abc123

📤 PROMPT:
שלח מייל ללקוח עם עדכון הזמנה

📥 RESPONSE:
✉️ נשלח מייל אוטומטי ללקוח

======================================================================
```
