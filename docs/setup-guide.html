Installation Steps

Install Python 3.7+

Choose one of these methods:

Option A: Homebrew (Recommended)

# Install Homebrew (if not installed) /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # Install Python brew install python3

Option B: Official Installer

Download from python.org

Verify Installation:

python3 --version # Should show: Python 3.7.x or higher

Clone/Download the Project

# Clone from repository git clone https://github.com/natovichat/cron-agent.git cd cron-agent # Or download and extract ZIP

Run Setup Script

# Run the simple setup command ./setup

This will automatically:

  • ‚úì Check Python version (3.7+)
  • ‚úì Create virtual environment in src/venv/
  • ‚úì Install all dependencies
  • ‚úì Create .env file from template
Takes ~30 seconds. All technical files stay in src/!

Configure Todoist API Token

Edit the .env file (in root directory) and add your Todoist API token:

# Open .env file nano .env # Add your token TODOIST_TOKEN=your_token_here

Get your token from: Todoist Settings ‚Üí Integrations ‚Üí Developer

Optional: You can also adjust settings in config.json (polling interval, log directories).

Install Scheduler (LaunchAgent)

# Install and start the scheduler ./cronagent --install

This creates and loads the LaunchAgent that will run your cron agent every 5 minutes.

Note: Default interval is 5 minutes. To change it, edit config.json and reinstall.

Verify It's Running

# Check scheduler status ./cronagent --status # View technical logs tail -f logs/stdout.log # View conversation logs (Cursor AI interactions) ls -la clean_logs/ cat clean_logs/conversation_*.log
Success! Your cron agent is now running automatically every 5 minutes via LaunchAgent. All code is in src/ - you only need .env and config.json!

Management Commands

# Check status ./cronagent --status # Uninstall scheduler ./cronagent --uninstall # Manual run (for testing) ./cronagent # View LaunchAgent status launchctl list | grep cronagent
Sleep Mode: If your Mac goes to sleep, the LaunchAgent will automatically catch up when it wakes. This is more reliable than traditional cron!

Scheduler Details

macOS LaunchAgent

  • ‚úÖ Most reliable - Apple's recommended method
  • ‚úÖ Catches up after sleep
  • ‚úÖ No admin rights needed
  • ‚úÖ Runs on login
  • üìÑ Config: .plist XML file

Linux systemd (Preferred)

  • ‚úÖ Modern standard on most distros
  • ‚úÖ Catches up missed runs
  • ‚úÖ User-level timers
  • ‚úÖ Better logging
  • üìÑ Config: .service + .timer files

Linux cron (Fallback)

  • ‚ö†Ô∏è Traditional, works everywhere
  • ‚ùå Doesn't catch up after suspend
  • ‚úÖ Simple and reliable
  • ‚úÖ Works on all Linux distros
  • üìÑ Config: crontab entry

Windows Task Scheduler

  • ‚úÖ Built into Windows
  • ‚úÖ Catches up missed runs
  • ‚ö†Ô∏è Requires admin for install
  • ‚úÖ GUI available (taskschd.msc)
  • üìÑ Config: XML task definition
