Installation Steps

Install Python 3.8+

Choose one of these methods:

Option A: Homebrew (Recommended)

# Install Homebrew (if not installed) /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # Install Python brew install python3

Option B: Official Installer

Download from python.org

Verify Installation:

python3 --version # Should show: Python 3.8.x or higher

Clone/Download the Project

# Clone from repository git clone https://github.com/yourusername/cron-agent.git cd cron-agent # Or download and extract ZIP

Run Setup Script

# Run the cross-platform setup python3 setup.py

This will:

  • ‚úì Check Python version
  • ‚úì Create virtual environment
  • ‚úì Install all dependencies
  • ‚úì Create .env file from template
  • ‚úì Create LaunchAgent plist

Configure Todoist API Token

Edit the .env file and add your Todoist API token:

# Open .env file nano .env # Add your token TODOIST_TOKEN=your_token_here

Get your token from: Todoist Settings ‚Üí Integrations ‚Üí Developer

Install Scheduler (LaunchAgent)

# Install and start the scheduler python3 cron_agent.py --install

This creates and loads the LaunchAgent that will run your cron agent every 5 minutes.

Verify It's Running

# Check scheduler status python3 cron_agent.py --status # View logs tail -f logs/stdout.log # View clean logs ./view_clean_logs.sh
Success! Your cron agent is now running automatically every 5 minutes via LaunchAgent.

Management Commands

# Check status python3 cron_agent.py --status # Uninstall scheduler python3 cron_agent.py --uninstall # Manual run (for testing) python3 cron_agent.py # View LaunchAgent logs launchctl list | grep cronagent
Note: If your Mac goes to sleep, the LaunchAgent will automatically catch up when it wakes. This is more reliable than traditional cron!

Requirements

‚úÖ
Python 3.8+ Required

Must be installed manually

‚úÖ
pip Required

‚öôÔ∏è
systemd (preferred) Optional

Built into modern distros (Ubuntu 16+, Fedora, Debian 8+)

‚è∞

Smart Detection: Setup script automatically detects if systemd is available and uses it. Falls back to cron if systemd is not present.

What Gets Installed?

  • üì¶ Python virtual environment (venv/)
  • üìö All dependencies from requirements.txt
  • ‚öôÔ∏è Configuration file (.env)
  • ‚è∞ systemd: Timer + Service files (~/.config/systemd/user/)
  • ‚è∞ cron: Crontab entry (user crontab, no sudo)
  • üìù Log directories

Installation Steps

Install Python 3.8+

Ubuntu/Debian:

sudo apt update sudo apt install python3 python3-pip python3-venv

Fedora/RHEL/CentOS:

sudo dnf install python3 python3-pip

Arch Linux:

sudo pacman -S python python-pip

Verify Installation:

python3 --version pip3 --version

Clone/Download the Project

git clone https://github.com/yourusername/cron-agent.git cd cron-agent

Run Setup Script

# Run the cross-platform setup python3 setup.py

This will automatically detect your system and install the best scheduler:

  • ‚úì systemd available? ‚Üí Install systemd timer (modern, recommended)
  • ‚úì No systemd? ‚Üí Install cron entry (universal fallback)

Configure Todoist API Token

# Edit .env file nano .env # Add your token TODOIST_TOKEN=your_token_here

Get token from: Todoist Developer Settings

Install Scheduler

# Install and start the scheduler python3 cron_agent.py --install

For systemd (if detected):

  • Creates ~/.config/systemd/user/cronagent.service
  • Creates ~/.config/systemd/user/cronagent.timer
  • Enables and starts the timer

For cron (fallback):

  • Adds entry to user crontab
  • Runs every 5 minutes

Verify It's Running

For systemd:

# Check timer status systemctl --user status cronagent.timer # Check service status systemctl --user status cronagent.service # View logs journalctl --user -u cronagent.service -f

For cron:

# View crontab crontab -l # View logs tail -f ~/cron-agent/logs/stdout.log
Success! Your cron agent is now running automatically every 5 minutes.

Management Commands

systemd:

# Check status python3 cron_agent.py --status systemctl --user status cronagent.timer # Stop timer systemctl --user stop cronagent.timer # Start timer systemctl --user start cronagent.timer # Uninstall python3 cron_agent.py --uninstall

cron:

# Check status python3 cron_agent.py --status # Edit crontab crontab -e # View crontab crontab -l # Uninstall python3 cron_agent.py --uninstall

Requirements

‚úÖ

Download from python.org or Microsoft Store

‚úÖ
pip Auto

Admin Privileges: Task Scheduler installation may require "Run as Administrator" depending on your Windows configuration. If setup fails, try running Command Prompt or PowerShell as Administrator.

What Gets Installed?

  • üì¶ Python virtual environment (venv\)
  • üìö All dependencies from requirements.txt
  • ‚öôÔ∏è Configuration file (.env)
  • ‚è∞ Windows Task Scheduler task (runs every 5 minutes)
  • üìù Log directories

Installation Steps

Install Python 3.8+

Option A: Official Installer (Recommended)

  1. Download from python.org
  2. Run installer
  3. IMPORTANT: Check "Add Python to PATH"
  4. Check "pip" option
  5. Click "Install Now"

Option B: Microsoft Store

  1. Open Microsoft Store
  2. Search "Python 3.11" (or latest)
  3. Click "Get" to install

Verify Installation:

# Open Command Prompt or PowerShell python --version pip --version

Download the Project

Download and extract the ZIP file, or clone with git:

git clone https://github.com/yourusername/cron-agent.git cd cron-agent

Run Setup Script

Open Command Prompt or PowerShell in the project directory:

# Run the cross-platform setup python setup.py
If you get a permission error, try:
  1. Right-click Command Prompt ‚Üí "Run as Administrator"
  2. Navigate to project folder: cd C:\path\to\cron-agent
  3. Run: python setup.py

Configure Todoist API Token

# Open .env file with Notepad notepad .env # Add your token TODOIST_TOKEN=your_token_here

Save and close Notepad.

Get token from: Todoist Developer Settings

Install Windows Task Scheduler Task

# Install and start the scheduler python cron_agent.py --install

This creates a scheduled task named "CursorCronAgent" that runs every 5 minutes.

Verify It's Running

Via Command Line:

# Check status python cron_agent.py --status # View task in Task Scheduler schtasks /Query /TN "CursorCronAgent" /V /FO LIST

Via GUI:

  1. Press Win + R
  2. Type: taskschd.msc
  3. Look for "CursorCronAgent" in Task Scheduler Library
  4. Check "Last Run Time" and "Next Run Time"
Success! Your cron agent is now running automatically every 5 minutes via Windows Task Scheduler.

Management Commands

# Check status python cron_agent.py --status # Manually run task schtasks /Run /TN "CursorCronAgent" # Stop task schtasks /End /TN "CursorCronAgent" # Uninstall scheduler python cron_agent.py --uninstall # View logs type logs\stdout.log

Alternative: Run Manually

If you don't want to use Task Scheduler, you can run the agent manually:
# Run once python cron_agent.py # Or keep a Command Prompt open and run periodically

Recommended Use Cases

Use macOS LaunchAgents if:
  • You're on a Mac (obviously!)
  • You want the most reliable laptop scheduling
  • You prefer Apple's ecosystem approach
Use Linux systemd if:
  • You're on a modern Linux distro (Ubuntu 16+, Fedora, etc.)
  • You want advanced scheduling features
  • You prefer modern service management
Use Linux cron if:
  • You're on an older Linux distro without systemd
  • You prefer traditional Unix tools
  • You need maximum compatibility
Use Windows Task Scheduler if:
  • You're on Windows (obviously!)
  • You want GUI management
  • You need enterprise integration
